CI/CD 란

Continuous Integration와 Continuous Devliery 또는 Continuous Deployment 의 약어로, 소프트웨어 개발 및 배포 프로세스를 자동화하고 개선하는
방법을 가르킵니다. 

CI 프로세스는 주로 다음과 같은 단계를 포함합니다.
- 코드 작성 (자바 작성)
- 자동 빌드 (Gradle 설정)
- 자동 테스트 (빌드 후에 통합 테스트 코드를 실행 JUnit)
- 자동화된 통합 (빌드 및 테스트 성공 후 통합 브렌치에 자동 머지)

CD는 다음과 같은 단계를 포함합니다.
- CI를 통한 자동화 (젠킨스, Gitlab CI등 코드가 변경될때마다 개발자에게 알람을 준다.)
- 자동 배포 준비 (배포를 위한 아티펙트(artifact) 생성 )
- 수동 승인 (QA, 팀장 매니저 권한이 있는 사람이 배포 승인하는 단계)
- 자동 배포 (마지막 단계, docker , kubernetes, ansible)

아티펙트(artifact): 최종 산출물을 의미한다. 컴파일된 .jar .war 파일이 될 수 있다.

Plan > Code > Build > Test > Release > Deploy > Operate

Gradle 이란
Gradle JVM 기반 프로젝트를 빌드하고 관리하기 위한 오픈 소스 빌드 자동화 플러그인 시스템
다양한 플러그인을 제공하며, 이를 통해 다양한 프로젝트 유형을 지원합니다.

의존성 관리: Gradle은 프로젝트의 의존성을 관리
멀티 프로젝트 빌드: Gradle은 여러 프로젝트를 하나의 빌드로 관리할 수 있습니다.
빌드 캐시: 빌드 시간을 최적화하기 위해 Gradle은 이전 빌드에서 생성된 산출물을 캐시하고 빌드 시간을 단축시킵니다.
유연한 테스트: Gradle은 테스크 단위로 빌드 작업을 정의하여, 이를 활용하여 사용자 정의 빌드 프로세스를 만들 수 있습니다.
